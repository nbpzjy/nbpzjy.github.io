hexo 运行有几年了，最初跑在 gitbub 上的 pages，自己的域名，现在也还在，但是也同时原样同步到自己的群晖服务器上了——这一步是自动的。反正为了外网访问群晖，已经 DDNS 了，多跑一个 web 服务也不耽误什么，除了因为 80、443 端口被封锁，只能自己定义一个端口进行转发，需要带端口访问之外，一切都很完美。

本来这次是打算迁移到 hugo 系统去的，研究了半天，搭建是成功了，主题也很漂亮，渲染也快，而且还可以实时预览，很赞，但是模板配置太麻烦，主要是想到以前的 hexo 下的文章模板不同，设置起来实在坑爹，测试迁移了几个，发现需要手动修改，实在费劲。就放弃了。

虽然放弃了，但是也有收获：

那就是发现了一个新的博客评论系统。

很久以前用过多说，可惜已经死掉了——默哀。又没有特别需要这个评论功能，看到很多有关静态博客评论的，都是要么太丑要么带广告，好多还需要填太多东西，不支持匿名评论，都懒得去搞了。但是现在发现了一个牛逼的静态博客评论系统 valine，基于 leancloud 来的，简单又漂亮，这种风格，实在喜欢，所以忍不住去搞一下。

为了能食用 valine 评论系统，需要做几件事：

  • 升级当前主题到支持 valine 的版本
  • 在 leancloud 注册并新建一个应用
  • 配置好相关文件

主题升级到新的版本

为什么要升级呢？

因为自己从零开始去设置 valine 实在是麻烦，而新的主题以及直接支持了 valine 评论，所以只要升级好了，做一些设置就完成了。

配置相关文件

重点来了

  1. 首先是设置 valine 支持

这个容易，直接在主题里面的 config 文件里面修改 valine 为 true，并填写好上面得到的 App ID 和 App Key，就可以开始 hexo g 渲染了，基本上没什么问题，能成功出现评论，激动不？

但是注意到了么？评论区的右下有一个 powered by valine，还有个版本号。

作为一个追求极致的人来说，有点忍不了。

  1. 干掉 valine 信息

怎么干掉这个内容，找了很多方法，也搜索了大半晚上，基本没找到靠谱的内容，但是又看到很多个人博客使用 valine 时都是干干净净的，却没有人遇到过怎么修改的问题。奇怪。

内容找到了，但是修改是个问题。先看两个图：

一个版本号是写死的，一个是自动获取的。经过测试，版本号写死的直接删掉就好了，而自动获取的删掉就出错。摸索了半天，也懒得去研究自动获取的错误到底来自哪里，索性就用老版本的 valine.min.js 来做，反正差别应该不到。经过试验，可以一直到 1.3.3 点这里可以直达 版本是直接写死的版本号，之后的都是自动获取的了。
