群晖中用 docker 部署 h5ai 文件服务器

May 15, 2019 | 网络技术 | 1.2k 字 | 3 分钟
条评论

最近在油管上闲逛，发现了一个好玩的东西叫 h5ai 的文件目录服务。这是一个部署很简单，但执行起来很高效的服务器。什么架构啊什么技术啊，就不多说了，直接使用就好了。这方便了外网时的访问，mac 或者 pc 不用再担心找不到合适的协议连接出问题，直接使用浏览器搞定一切。

为了简单方便，打算采用 docker 来部署，这样的好处是不用担心很多依赖问题，简单，高效。

群晖运行的 docker 跟普通的使用命令行运行的 docker 稍有区别，提供了一个可视化的界面，所以有一些配置，跟命令行相比有时是简单了，但有时又复杂了。一个命令可以解决的事，在可视化的界面上设置可能出现各种问题。

下载映像

运行 docker，在注册表里面输入 h5ai 搜索，找到 clue/h5ai 的这个，点击下载。文件不小，大约有 270 M，需要点耐心等待其下载完成。

运行映像

下载完成之后，就可以开始部署了。

点击映像，选择下载好的 clue/h5ai:latest，再点击启动。

容器名称。——这里顺便说一下，现在用的 docker 容器多了，就出现端口记不住的问题，学到一个新的方法，就是在容器命名时，把名字后面带上本地端口（也就是外部访问这个容器时的端口号），这样一看就知道该容器运行在什么端口上，管理方便，使用也方便，算是最近学到的一个小技巧。

高级设置，进入之后要做两件事：

  • 设置端口号，这个端口不要跟一句运行的其他容器搞冲突了就好，可以自动，但是自动的话总觉得不可控，最好还是自己添加一个喜欢的数字。顺便把这个端口号写到上面提到的容器名称那里去。

  • 还有一个重要的事情，就是卷的添加。这里需要多说说，因为我遇到的坑就在这里。添加文件夹，可以进入到群晖的目录里面去选择需要挂载到 h5ai 文件服务的目录，原则上可以选择所有的目录或者目录里面的子目录。然后重点来了：装载路径。在网上搜来搜去也没找到具体该填什么，后来在初始的 h5ai 里面的 readme.md 文件里面看到了，叫 /var/www ，可以在这个目录下新建一个对应的文件目录的名字，当然也不写，那样就直接挂载在根目录下面了。个人觉得还是写一个好些，反正又不要钱。然后这样就挂载好了。

你以为这样就成功了吗？
Too young too simple.

群晖共享文件的权限问题

这时打开是空白的，虽然新添加的目录确实出现了，但是里面是空的。 nani？明明里面有文件的啊，咋就啥都没有呢？我一度怀疑我没有挂载成功，只是添加了一个空目录。但是把之前专门给 docker 使用的目录挂载进来，却是成功读取到文件的。那么就是权限的问题了。

找到问题就好解决了。这个问题在其他平台上不知会不会遇到，但群晖是会遇到的。权限出问题，那就去需要挂载的目录添加权限：

回到群晖，打开 file station，找到需要共享的目录（前面已经挂载的目录），右键点击，再选择属性，弹出栏的第二个标签权限，新增一个 Everyone 的用户，把其权限改为读取（也可以完全控制，但最少要有读取权限），注意记得选择应用到子文件夹，不然后面还得出现空白的文件目录——别问我怎么知道的。

这样，再次运行已经设置好的容器，用浏览器访问群晖的地址加上 :端口号， 就可以访问到刚才做好的 h5ai 目录服务了。赞不赞？

结束

简单高效的文件服务就搞定了，外网也可以轻松访问小姐姐们了。虽然群晖有自带的 webDav 套件可用，但是一直没用过，不好评论，这个 h5ai 却是极好的，起码安装方便，目录可控。

越发觉得 Docker 容器是个好东西了。
