最近在蜗牛星际矿难的熏陶下，在同事一两酒童鞋的带动下，在咸鱼那方便快捷万物皆有的光芒下，以低廉的“车票”费用，搭上了矿难这趟车。对于爱折腾的人来说，不用解释，直接上车。与此同时也开始了 NAS 折腾之路。

路在于折腾，折腾多了，路就宽了。

矿难车的主要目的，就是奔着 NAS 来的。本来嘛，这样的矿渣车，也跑不了别的，由于带有四盘位的插槽——用来搭 NAS 简直是绝配。

经过多方了解，说是 D 款翻车的概率小点，单口又不存在什么双口的百兆千兆的问题，据说机箱还是 ABCD 几款中最好看的。所以，没有与一两酒童鞋的相同（他是 C ）。以 290 的价格，拜了一个 D 款单口。

来了就开始 NAS 之路：说干就干。

什么是 NAS

NAS，全称应该是 Network Attached Storage ，即网络存错设备。目前市场上的NAS有比较多的选择，其中群晖、U-Nas、铁马威、威联通等等常见的也是大家玩得起的，企业级的就不敢说了（也没关注过），而那些玩客云、路由器接移动硬盘什么的，也算是其中的一种方式，但是由于可玩性一般，所以暂时不说。

用的多的，应该就是那四种的。其中群晖应该是玩的比较多的，而且各种套件应该也是最齐全的——本来嘛，玩的人多了，自然玩法也就多了。各种黑白群晖，随便一搜都是一大把的教程、分享、问题等等。

而我的矿难车，自然要玩的是群晖了。具体安装方法以及其中的坑，就不在这里细说了，毕竟这里是说外网访问的问题的。

简单讲，NAS 就是一个网络存错设备。比如百度云、oneDrive、GoogleDrive 等的东西，只不过那些是别人家的服务器，说不定哪天就跟360 云那样拜拜了。而 NAS 是你自己家的，只要你家还有电还有网络，他就在，具有绝对的控制权，而且想要多大空间就买多大空间——只要你有钱。而起其可玩性，远比那些某某云的要多得多，比如 Docker 啦，比如 Moments 啦……

不过多说这个问题，什么感觉请入坑来体验。

什么是内网外网

我们上网，基本都是光纤进来，接入光猫，光猫再接入路由器，而路由器就可以提供 WIFI 信号给移动设备或者带 WIFI 的设备，同时也可以连接线缆提供上网。不管是哪种方式，我们都是躲在路由器下面的，我们所有连接在路由器上的设备们，不管有线还是无线，我们也组成了一个内网，所以我们的 NAS 也就是在这个内网下面的。

要想从外网——也就是不在我们路由器下面的联网设备上访问我们的 NAS ，就是所谓的外网访问内网的问题了，比如手机连接 4G 网络来访问就是此列。自然，我们暴露在外面的不是 NAS ，而是路由器，所以要访问 NAS 还需要路由器来帮忙：这就是路由器的端口转发。

什么是公网 IP

从外面访问，涉及到一个外网 IP 的问题。

是这样的，我们访问某个网站或者某个服务器，需要的是这个服务器或者网站所在的主机的IP地址。而我们普通人家的宽带，亦或是移动设备等的上网，是由网络服务供应商给的 IP ，而全球的 IP 资源都非常有限，我们绝大多数的电脑都是处在一级一级的路由器下面，我们需要访问的服务都是通过一级一级的上一层的服务器去获取再返回给我们的。而这个处在一级级路由器下面的我们，被提供给的 IP ，极大多数情况是一个内网的 IP ，也就是所说的私网 IP ——这就是我一开始采坑的原因，后面接着讲这个问题。

而我们需要从外面访问我们的 NAS ，就需要我们的路由器有一个公网的 IP ，也就是在全球互联网上能被访问到的这样一个 IP ，这个 IP 是全球都可以访问的被暴露在全球互联网上的一个 IP 地址——牛不？但是由于 IP 地址资源有限，所以一般人家，网络服务提供商，也就是移动联通电信等，是不会给一个固定的 IP 地址的，而是每次拨号上网都会是一个不同的 IP ，甚至一段时间之后也会改变，这个会改变的 IP 就是动态 IP ，尽管这个已经是一个公网 IP ，但是一个会变化的公网 IP 。想要一个固定的公网 IP 么？花钱。

什么是 DDNS

上面说过，服务商给我们的 IP 是动态改变的。

那么一个在变化的 IP ，怎么才能访问 NAS 呢？显然不可能每次都去查一下现在的公网 IP 是多少再去使用 NAS ，那也太糟糕了。

DDNS，动态域名解析。

这个就是为了对付这些老是变化的 IP 地址的。其实市面上也有很多的解决方案，来对付这种变化的 IP 地址，实现外网访问内网的方法，比如花生壳等。但是需要花钱，当然也有免费的，给一个难看的二级域名，而且速度垃圾。最妙的还是自己注册一个域名，然后 DDNS ，完美高效地解决这个外网访问内网，连接 NAS 的伟大任务。

开始外网访问之路

上面介绍了蛮多内容，对 NAS 有了一定的了解——至于安装，以后再说。也说到了内网外网以及公网IP，还提到了 DDNS ，下面就开始外网访问之路。

先说踩过的坑

开始之前，先说说踩过的坑。

由于 DDNS ，可以有多钟方式来实现：

  1. 路由器
  2. NAS 自带
  3. NAS 运行 Docker 来实现

其实也就是通过一种方式，把目前自己的公网 IP 发送给域名解析服务器，并修改域名解析服务器里面的 A 记录。这个过程说起来简单，实现起来还是挺麻烦的。首先需要能提供 DDNS 的设置的设备，不管是路由器还是群晖（NAS）本身，都有自己的调性，对自己熟悉的域名解析服务商不提供选择，那就无解。比如我的 Linksys ，只提供 NO-IP 和 DynDNS ,

而国内的域名服务商一个都没有，这两个是国外的访问性能不佳是其次，重点是竟然还收钱——真是不能忍。所以刚买的 Linksys 就成了一个拦路虎，把在路由器上设置 DDNS 的路挡得死死的——要么放弃要么换路由器，可是这玩意才为了即将升级到 200M 的电信宽带而买来的双千兆路由器，难道就这么放弃？显然不能这么任性。不换路由器，那就想其他办法。

也想过插一个之前的路由器，专门为了 DDNS 而插一个空的路由器。毕竟 DDNS 只需要设置完成并由这个设备把IP地址信息发送到 DDNS 域名解析服务商并成功更新 IP 地址，即完成其任务了，所以单单独连接一个路由器并保持为路由转发或者叫 AP 模式，也就是不单独组网为下游提供服务，而是在目前路由器的旁边，不需要获取IP地址地接入网络，为 DDNS 提供 IP 地址更新信息的发送服务。可惜把我当小米路由器 R3 打开看看，只有花生壳一家。如果想用那么得刷路由器，难度应该不大，但现在重点是解决 DDNS 问题。所以还得想别的办法。

开心的是，群晖的 DSM 设置页面的外部访问，提供了设置 DDNS 的方法。群晖自己可能也希望用户自己解决外网访问的问题，毕竟 QuickConnect 是要消耗其服务器资源的。并且这个 DDNS 设置提供了毕竟丰富的选择，其中 DNSPod.cn 是国内用的比较多的，马化腾家的。

不开心的是，这个设置完成之后，一直提示主机名称找不到，几乎折腾的人到达崩溃边缘。各种办法都想了，度娘和谷歌都搜了一大堆，问题基本上没得到解决。经过深思熟虑，并咨询一两酒童鞋，认为应该是IP地址的问题。一直以为自己应该是公网IP吧？但是一直无法成功把这个 IP 地址更新到 DNSPod 上去。

首先是度娘搜索 IP 得到的对外 IP 无法 PING 通。这就完蛋了，说明没有物理连接啊。这就是说即使 DDNS 成功了，解析到这个地址，还是无法访问的。这个应该就是私网 IP 的痛点吧？

后来联系电信，确实不是公网 IP ，申请更改为公网 IP ，对方说没有问题，大溉 24H 生效。实际上不到 24 小时就生效了。然后度娘查到的 IP 也能轻松 PING 通了，设置好路由器的转发，用路由器也能轻松地访问到 NAS 了，简直是太棒了。

然后是继续折腾 DDNS 的问题。问题是依然找不到主机名。

域名申请

上面提到了 DDNS 中的顶级域名，这是要花钱的。国内提供域名注册的还不少，花生壳也有，但是价格贵。马云家也有，马化腾家也有。通过对比，马化腾家的价格最低，而且重点是现在的 DNSPod 也是他家的，所以自然域名也在他家注册了。

公网IP申请

需要外网能访问，最大的一个重点就是要有公网 IP ，不管是固定的还是动态的。有固定的最好，连 DDNS 都省了，但是得花钱，个人用户没必要去占用固定的 IP ，主要是没必要花钱，更重要的是有 DDNS 可以解决。所以，只要是公网 IP ，不管固定还是动态，足矣。

申请公网 IP 不难，我指的是电信网络，至于其他的，没用过不了解，但据说移动宽带是不提供公网 IP 的，直接放弃吧，要么换其他的网络。

电信申请公网 IP ，打 10000 ，保修或者别的，反正需要人工服务，然后说自己买了网络硬盘需要外面访问，希望开通公网 IP ，基本没什么问题，之后就坐等 IP 变为公网 IP ，话务员是说 24 小时之内再试试，但实际上没几个小时就已经是公网 IP 了，完美！

域名解析

公网 IP 有了，域名也申请了，就开始 DDNS 了。

上面采坑里面提到，路由器设置的路没通，旁边挂路由器也没通——得刷路由器固件，那样的话还得折腾路由器，而且那样的话还不如直接把Linkysy刷了得了。由于目前还有群晖自带的外网访问这条路可以走，先探探这条路能走多远，不行再研究刷路由器的事。

上面的坑踩到的，其中一部分是把私网IP当成公网IP来用，自然是失败了。现在已经确认是公网 IP 了，再设置。

DNSPod.cn 提供了 DDNS 服务，而群晖自带的 DDNS 设置里面也有 DNSPod.cn 可以选择，简直是完美。思路很简单：

首先在 DNSPod.cn 中把之前申请的域名加入，并添加一个解析信息。由于是在马化腾家申请的域名，所以直接点击域名解析就有了这个域名，进去之后发现已经有了连个信息，是关于免费的域名解析的，记录类型是NS，需要做的是点击添加记录，主机记录里面填 www ，类型选择 A ，后面填入当前的IP地址，这样一不会犯错，二也保证在 DDNS 提供修改 IP 信息之前已经是正确的 IP 了，当然随便填一个 123.123.123.123 也没问题，反正这个 IP 是要被 DDNS 修改的。

再在用户控制中心的安全设置中获取 API Token 。这个玩意在群晖的外网访问中就是用户名和邮箱。点击创建 API Token ，会要求输入一个名字，代表这个 API Tolen ，之后生成
，得到一小串数字，和一大段数字字母组合。这个就是需要填在邮箱和密码那里的信息了。友情提示：上面这个 API Token 已经被我删掉了，只是作为例图。

可是，在外网链接里面设置完成，点击联机测试，刷新之后任然是可怕的红色的找不到主机名。也就是说还是出问题了。难点是，这回想象不出问题出在哪里了。

完美解决

域名解析，设置完成，外网访问的 DDNS 设置也完成，但是测试仍然是失败，仍然是找不到主机名，真是折腾人。

一度认为，真的要去折腾路由器刷固件么？

灵光一闪：有没有可能是因为 .cn 结尾的问题？是不是测试一下就知道了。

正好之前的 .com 域名还在用，用作博客了。试试能不能走得通。由于 .com 是在马云家注册的，所以这个还跟 .com.cn 结尾那个不太一样，需要再马云家修改 DNS 服务器。修改这个 DNS 服务器要去马云家的域名控制台，点击域名，然后在这个域名的最后面点击“管理”
，然后在 DNS 修改中把马化腾家说好的两个免费DNS地址输入
，保存，就完成了，并且由于之前用作博客了，现在要换给 DDNS 测试，所以把解析信息也都暂停，不用删除——万一失败了还得用回博客，不用再设置，点击启用即可。

再回到马化腾家的控制台，把 .com 的域名加入进来，按照之前的 .com.cn 那个方法，添加一个A记录，也是 www，以及 IP 地址。

回到群晖的外部访问中，修改为 .com 的主机信息，由于 API Token 没变，所以都不用修改，直接点击联机测试——我的天，竟然显示绿色的成功。这真的是太棒了。

之后就是在路由器设置端口转发，外部端口自己指定一个，转发到群晖的 IP 地址，端口 5000 ，就可以用自己的域名后面带刚才指定的端口访问到内网中的 NAS 了。其他的比如 Web 服务、或者图床服务等之后再折腾了。

访问也一切正常了，世界一下子变得美好，窗外的鸟鸣也更清脆了。

这个坑有点坑

这个坑，说的就是 .com.cn 的域名，不知道问题出在哪里，群晖的外部访问就是无法找到主机名，直接无解。好在测试了 .com 的域名，并且那么容易就成功了，显然跟这个 .cn 结尾是有关系的。要说是备案问题，那么将这个 .com.cn 的 IP 换给博客，就直接成功了，指向的是 github 的 pages 地址。也就是说，群晖对于 .com.cn 的访问时有问题的，问题到底在哪里呢？是在群晖这边呢？还是 DNSPod.cn 呢？或者是在域名本身呢？

记得之前用一个类似的 .cn 的地址指向一个 vps 准备用来搞一个 ownCloud 做网络存储什么的，就出现一个备案问题。极大一个原因，是用 .com.cn 的域名指向某一个 IP 地址的话，是需要网站备案的，而指向另外一个域名则不用备案——暂且就这么理解吧。反正是一个坑，一句话就是如果打算用来做 DDNS 的域名，最好还是不要 .cn 结尾的了，其他的不知道情况怎么样，至少 .com 的不会出问题，一两酒童鞋的 .top 域名也没有问题。

总结一下

归纳起来，想要外部访问，就需要公网 IP ，能固定最好，连 DDNS 都省了，但是一般没有固定公网 IP ，那么就准备 DDNS 。路由器能设置也很完美，不行还有群晖的外部访问设置这条路。而域名申请，要注意一个问题，不要.cn结尾的，省的麻烦。

条条大路通罗马，解决的方法也很多。祝 NAS 开心，另外在想：是不是也要 UPS 呢？那个垃圾 16G 固态要不要扔了？还有内存要不要换？坑里的生活很精彩，比如 Plex 订阅已经干掉 880 米，而域名又干掉 160 米（ 5 年的 .com.cn ，虽然后来也没用上），一个希捷酷狼又是 890 ……入坑愉快……
