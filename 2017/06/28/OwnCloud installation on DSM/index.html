背景描述

最近用一个老旧的三星上网本安装了黑群晖，准备尝尝群晖的味道，以备以后有钱的时候拜一个回来打下基础，顺便把已经处于废弃状态的上网本利用起来。

对于安装黑群晖，等有时间再写个文章好了，现在就不在这上面浪费时间了。毕竟这个是专门为了安装OwnCloud而写的一个博文。

获取OwnCloud安装包

虽然网上可以找到很多很多的安装包，但是去官网还是比较靠谱的一个做法。
https://owncloud.org/，最新的版本是10.0.2(可以点击下载最新的这个版本)。

下载的版本也要注意，如果在普通电脑上安装，选最新版本就好了，不同的版本对于不同版本的PHP有要求。第一次尝试就是下载新版本，但是最新的版本要求PHP5.6，而DSM的PHP版本是5.5.16，所以直接就无法运行。不得已又下一个较旧的版本9.0.10，这是没有其他好办法的办法——如果可以升级DSM的PHP版本，那就不用老的owncloud版本了。怎么升级PHP，这是下一个问题，至少目前没哟找到升级PHP的方法，毕竟这不是普通的电脑，但是我想直接在上网本上用命令行或者能升级，由于对群晖系统不是很了解，不知道升级PHP会造成什么样的后遗症，暂时还是将就PHP而选择低版本的OwnCloud了。

安装

解压上面的安装包（zip格式），之后把解压后的文件拷贝到群晖上的web文件夹下面，就算完成了。

上面说的很简单，虽然确实不难，但是在之前需要设置DSM的网络服务，也就是web service，只有设置成功了之后才能有web文件夹。如果需要安装phpAdmin，则需要在之前安装好MariaDB。

第一次尝试失败

第一次安装时用的最新版本，所以运行的时候提示需要PHP5.6，而我的DSM上的PHP5.5.16，失败。

第二次安装

吸取上一次的失败，重新选择了9.0.10版本的OwnCloud，这是最老的一个还在支持的版本，也不知道具体的PHP版本的要求，但估摸着应该可以。

文件读写权限导致错误

安装之后运行,又出错了。

配置文件无法写入目录，显然是权限问题导致的失败。对了，对文件的读取写入权限没有设置，当然会出错。

设置文件夹读写权限

这里只在web目录下，所以需要设置的读写权限也就是web的读写权限。

  • 右键选择web之后点击Properties

  • 只需要对http的用户编辑许可就可以了，下图是编辑完成之后，获得了读写的权限，在编辑之前只有读的权限。

  • 这里就是修改权限的地方了，勾选之后OK就搞定了。

可以跑起来了，但是还是有个错误。

应该是DSM上是32位的PHP系统，而OwnCloud中的open_basedir设置需要64位的支持，要么把系统改为64位，要么在php.ini中修改open_basedir的设置。估计不是特别大错误，暂时不管，反正能运行就好了。

登录

用户名和密码都是在MariaDB设置的，也就是数据库的密码，没什么好说的。
