hexo 又又又采坑了

问题是这样的，有几个月没有更新博客了，主要是由于国家大会的召开，让我的github访问不了了，番羽墙也出不去，再加上要学习，就有很久没有折腾这事了。默认为github被墙了。

最近想放一篇学习时写的算是心得的东西上去，却怎么也搞不定。而github明明已经可以正常访问了啊、

权限问题


hexo的生成和预览都没问题，而部署命令却出现了被拒绝的提示。

  • 首先是没有权限访问
    • 这个应该是文件夹的权限，只能sudo来解决
    • 但是sudo之后就遇到了下一个问题
  • 然后是远程没有权限
    • 这个貌似是ssh的连接问题，但是查看ssh之后一切正常
    • 并且除了在hexo下遇到这个问题之外，其他的push或者是clone命令都正常的很

解决之路

知乎上有人说

看样子你应该是 备份了 博客目录, 之后在别的电脑上重新使用的时候出现的这个问题.
解决方法:
删除 博客目录下的 .deploy_git 文件夹.
之后在 hexo d 试试.

可是我一直找不到.deploy_git在哪里，也就不知道怎么删除了——搜索也没解决问题，也许别人都知道而且是个极为简单的问题，可是我就是找不到。

然后也有人说是bublickey不是在root账户生成的原因。

随即立即验证这个推测。
1、为root生成publickey:sudo ssh-keygen -t rsa -C “idealife@github.com”，对应路径为/var/root/.ssh
2、提取公钥信息并配置到github中。问题又来了，在UI界面中是无法访问到/var下的root文件夹的。我们可以通过sudo cat /var/root/.ssh/id_rsa.pub可以绕开这个问题。
3、执行sudo hexo deploy，部署成功！

那么就按照这个方法来生成新的sshkey，并添加到github上去。这下是可以成功hexo d了。但是有一个问题，必须使用sudo hexo d这个命令才行，一旦换回hexo d还是出现权限问题。这让我很不爽啊。

后来想到了查看.deploy_git的方法：

在hexo的文件夹下运行 ls -all
果然看到了.deploy_git文件夹——由于其实隐藏的，所以只能这个办法来查看。于是按照知乎上的解决办法，把.deploy_git这个文件夹删除。

第一次尝试：

一堆的权限拒绝——这个应该是rm的权限不够。于是：
执行sudo rm -rf .deploy_git，这下一下子就搞定了。

再查看hexo文件夹下是不是还有deploy_git文件夹呢：

重新hexo g之后再执行hexo d，这下终于成功了——注意是hexo d 而不是 sudo hexo d 哦

至此，搞定这个坑

虽然采用添加一个root的sshkey方法也可以解决，但显然不优雅——那就是每次都需要sudo hexo d 了，烦人的很。

重点是：

  1. 查看并删除隐藏文件
  2. 多思考总有解决办法
  3. 搜索不一定能解决——也许别人还没遇到这个坑

To be a better man.
