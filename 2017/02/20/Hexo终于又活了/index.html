Hexo终于活了

昨天搞到半夜4点半，竟然没搞定，总是出问题。如之前一篇文章所说，hexo g 生成文件，hexo s 启动本地服务预览，都没有问题，问题就是在部署到github上的时候，出错，总是说我的不是一个仓库。不得已就睡了，搞得今天上班都晕乎乎的。难过的是没搞定。

重新从头开始搞

昨天没搞定，一直想到底是什么原因失败?

问题出在部署的时候说我的不是仓库，谷歌之后有人说要把根目录下的.deploy.git删除重新来，我却怎么也找不到到底根目录下的这个文件夹在哪里。

![][image-1]
没办法解决。

后来想到可能是我拷贝以前的文件夹的原因

我这次是直接把以前的文件夹拷贝过来，之后直接在这个文件夹安装hexo，npm install依赖安装，估计这个文件夹的隐藏文件带过来了，虽然在生成html并且本地预览的时候没什么问题，但涉及到部署到服务器，就出错了。

重建文件夹终于搞定

今天直接删掉之前的文件夹，新建一个空白的，全新开始。虽然也尝试了几次才成功，但总算是搞定了。

安装hexo成功

这个看起来应该是失败的，这个也就是配置使用git推送部署到github的依赖，反正后面hexo d 失败的时候可以再安装一次。问题还是有，反正就是很多WARN，先不管了。

这个好复杂的样子，看起来也很牛逼，重点是最后那一句 INFO Start blogging with Hexo!

也不用看懂，反正就是可以开始blog了。看样子虽然只是执行了hexo init，貌似所有的依赖都安装完成了。这里试过几次，也失败了几次，主要的就是要科学上网，不然的话有可能还是会出问题的。

可以开始blog了

先把配置文件和主题文件拷贝过去——这样就可以把以前的配置文件用起来，也省的再去设置各个参数，包括部署到github都懒得改了，而且主题也过来了。
把之前的md文件拷贝到_post里去_然后执行hexo g

不用看懂，反正没错误，执行成功，也就是html文件生成OK，接下来就是预览加部署了。

很多东西，反正最后那一句是重点 INFO Deploy done: git

总结

Hexo部署博客真是踩了很多坑，算起来从之前的thinkpadR61e，到macbookpro，再到现在，搞了三四次了，每次都有不同的坑。反正乐趣就在折腾。
现在还有一个问题，当部署到github时，会把远程仓库里的CNAME这个文件删除，导致自己的域名访问失败，不过应该有解决办法，看是在本地建一个才cn name的文件在部署文件夹在推送的时候一起每次都推过去呢，还是怎么搞。先留着这个问题之后再搞了，昨天弄得太累。好在今天搞定了
