DaVinci出现RED Decoder错误

Feb 21, 2017 | 技术 | Hits

先上图

之前安装好的DaVinci12.5，版本是Resolve，iMac运行正常，并且学习了一级调色（何清超的初级教程）。然而今天竟然出问题了，在登录之后出现一个错误：
Failed to initialize RED Decoder with the configured number of decoders.
虽然在点击OK之后会进入工程，似乎各种操作也没什么问题，但是这样每次打开都出错也让人很无语啊

首先当然是祭出Google搜索大法

我也学会了在能不用度娘的情况下尽可能不用，最好google。直接原文粘贴过去搜。得到的搜索结果也蛮多。

解决方法

  1. 有说要买正版的——关键是Resolve版本本来就是正版啊，只不过属于免费版本，又没有做什么破解一类的。这个办法让我无语。

    MAC系统达芬奇提示错误的解决办法
    Failed to initialize RED Decoder with the configured number of decoders.
    Error: Decoder client version (11.1.0.023) does not match RED Decoder version (11.1.2.013)
    mac系统出现以上错误 是因为没能正常启动 编码解码 导致的，是编码器问题， 引起这种问题的 原因是使用了破解版的 达芬奇软件导致编码无法启动，请购买正版 插入加密狗可解决

  2. 有说下载工具的，可是我并没有什么下载工具啊。——后来才想起来我的百度云在下载东西，此事后话。

  3. Q:遇到错误提示“ Failed to initialize RED Decoder with the configured number of decoders”怎么办？装了REDCINE-X也不管用。关闭警告后好像不影响使用。
    A:这么看来应该不是REDCINE-X的问题了。这个问题在BMD官方论坛上也讨论过。有的人说重装达芬奇，有人说新建工程再试试。有人说是IP端口冲突造成的。官方说请上传达芬奇的LOG文件供他们分析。因此目前还没有一个定论。英文好的可以参考这个链接：http://forum.blackmagicdesign.com/viewtopic.php?f=3&t=528
    另外，如果你是Windows系统，防火墙可能会干扰DpDecoder通信。可以尝试关闭防火墙。看看是否问题解决了。另外，如果你使用BT下载软件，例如Utorrent之类，也可能干扰通信，关闭之后再试。
    附上下载地址：REDCINE-X_PRO_Build_33【Windows版】
    http://pan.baidu.com/s/1sjmICWl

总体来讲都还没解决

而这里却能找到解决办法。

重点在执行 sudo lsof -iTCP: 10000 -sTCP: LISTEN
这里是查看10000这个端口是什么程序在监听
看，有两个叫BaiduNeted PID 2151的在监听这个端口，而这个就是问题的关键了，百度云的端口跟RED Decoder的端口冲突了，到至了上文提到的那个错我。解决办法是停止百度云——就这么简单。停止百度云之后再查看这个端口是没有程序在监听的。然后打开DaVinci就正常了，没有上文的那个错误了，并且再去查看10000端口，是一个DPDcoder PID10087的程序在监听这个端口了
