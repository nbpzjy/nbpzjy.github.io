Hexo新建页面踩坑记

开始用Github+hexo部署静态博客有一小段时间了，虽然没怎么写文章，但是部署是成功的。之前是在windows上部署的。最近发现一个叫iPic的图床工具，对于Markdown这样的标记语言的图片展示困难户来说，简直就是救星啊。可惜只找到Mac版的，虽然也看到一个牛人自己写了Mpic作为windows版本的图片上传工具，可惜没有exe文件，自己编译计算了——其实是不会。
所以打算由windows转到mac来写博客的东西。

本来以为直接copy到优盘拿过来就可以了，不过出了很多问题。猜想应该是直接在优盘里不太行吧？本地预览是可以的，没有问题，写文字，generate，都没有问题，所有命令都完美。但是hexo d，就是运行不成功。尝试很多方法，就是推送不出去。npm install， hexo init，都搞了好几遍。没用。最后是windows上面把hexo的源文件整个推到github，再在mac上把它clone下来。问题还是接踵而开。

配置文件及主题文件

不知道怎么回事，配置文件和主题文件都没有过来。猜想大约是gitignore里面设置的问题，这两个东西根本没用推送到github上面去。

无所谓了。反正copy缺少的东西过去。就搞定了。

文章无法生成

这个就坑爹了
特么hexo new “xxxx”成功，然后在_post里面编辑相应文件之后，再回到shell执行hexo g，不行。

红红的error，真特么辣眼睛。
一开始没明白，怎么回事啊？后来发现，原来是特么tag：后面少了一个空格……

放上空格就搞定了。

hexo d的问题

之前在windows上已经很熟悉了。在mac上报一个错误

也是一个红红的error
不过这是小意思啦。不就是npm install hexo-deployer-git –save
就搞定了啊。

总结

hexo这玩意可以是可以，就是细节一定要注意，空格什么的，尤其在配置文件里面。少了一个空格，就不行了。最后希望多多写文章。

目前还只是写初级的东西。我的想法是，一部分引航相关的东西，另外有Android develop的相关东西，也会有一些生活杂事等等。主要是前面两点。
